## 2. 속도 체감 (16코어 PC, 워커 N=6)
| 작업 | 단일 스레드 | 워커 풀(병렬 벽시계) |
|---|---|---|
| 추천 1회 (1500표본, 알까기 후보 포함) | **~2,290 ms** | **~380 ms** |
| 추천 1회 (놓기만) | ~2,180 ms | (≈동일 비율) |
| 관찰 승률 (2500표본) | ~254 ms | **~38 ms** |

- **#3 효과**: 추천이 ~2.3초(단일) → **~0.4초**(6워커 병렬). 디바운스 200ms 더해 입력→추천 체감 **~0.6초**.
- **#1 효과**: 롤아웃 경량화(rolloutLevel 2·respLevel 2·expandShield off)로 한 판 비용이 옛 설정 대비 크게↓ (이 측정은 경량 설정 기준).
- 추천이 승률보다 ~9배 무거운 이유: 후보(≤3) × (쉴드보드 × 상대 6눈 × 표본)을 **후보마다 전수 롤아웃**하기 때문.

## 3. 로직 점검 — 정상 확인
- **선택 로직**(`finishMoveAdvice`): 승률 최상위 + ADV_TIE 이내 동률이면 알까기 우선 → 2라인 확정 놓기를 알까기보다 우선(버그 수정 반영). ✅
- **워커 집계**: 모든 워커가 같은 board·value로 `legalMoves` → 후보 키(`kind:line`) 일치 → rate 평균 정확. 워커마다 동일 표본수(per) → 단순 평균이 곧 전체 표본 평균. ✅
- **홀드 6단계**(`gradedHold`): 결정론적 락(확정 승/패) 우선 → 승률 권장/고려 순서 정상. ✅
- **승률 재사용(#2)**: 내 수 결정 시 추천이 낸 rate 재사용, 관찰/타짜/쉴드만 별도 MC. ✅
- **선공 첫 쉴드**: `shield` 인자가 워커까지 전달돼 첫 턴 평가 왜곡 없음. ✅
- **엣지**: 합법수 0(내 필드 가득)이면 scored 빈 배열 → advice null, 크래시 없음. ✅

## 4. 이상점 / 리스크 (버그는 아님, 개선거리)
1. **⚠️ 워커 무응답 시 멈춤** — 풀 집계는 N개 응답을 모두 기다림. 워커가 죽거나 응답 누락이면 `pending`이 안 끝나 "계산 중"에서 멈출 수 있음(타임아웃 폴백 없음). → **권장**: 일정 시간 내 도착분만으로 마감하는 타임아웃 가드.
2. **⚠️ 추천 비용(~0.4초/병렬)** — 후보를 전수 롤아웃해서 무거움. 더 줄이려면 (a) 싼 prior로 후보 가지치기, (b) 표본/후보 축소. 현재는 허용 범위.
3. **ℹ️ 스테일 요청** — 새 요청이 이전 것을 대체해도 워커가 돌던 작업은 취소 못 함(다음 작업이 큐로 대기). 디바운스로 완화되나 빠른 연속 입력 시 약간 지연.
4. **ℹ️ 기록(simLog) 지연** — 추천 도착(~0.4초) 전에 칸을 클릭하면 그 수의 `followedAdvice`/`advice` 기록이 비거나 직전 값일 수 있음(데이터 품질 약영향).
5. **ℹ️ 승률 의미** — 내 수 결정 시 표시 승률은 "추천 수를 두면 이길 확률"(직전 보드 승률 아님). 의도된 동작.
6. **ℹ️ expandShield off** — 알까기로 얻는 쉴드를 값 4 근사 1개로만 전개(6전수 대신). 속도 위해 의도된 근사, 정확도 영향 작음.
7. **ℹ️ 기존 lint** — 추천 요청 effect의 동기 setState(`set-state-in-effect`) 경고는 이전부터 있던 패턴(버그 아님).
