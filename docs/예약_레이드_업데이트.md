# 예약 레이드 업데이트 (주석 스테이징)

로스트아크 패치 일정에 맞춘 변경을 **미리 코드에 넣되 주석으로 비활성화**해 둔 상태.
레포는 정상 빌드되며, 그 사이 다른 변경을 자유롭게 배포해도 영향 없음.
**"A / B / C / D 배포해줘"** 라고 하면 해당 세트의 주석을 풀고 배포한다.

> 이 문서는 레포에 커밋되어 GitHub에 올라가므로, **다른 PC/세션에서 배포해도 기준이 따라온다.**
> 코드 안 마커: `UPDATE-A` ~ `UPDATE-D` (검색해서 찾기).

---

## 세트 요약

| 세트 | 시점(KST) | 내용 | 워커 재생성 |
|---|---|---|---|
| **A** | 6/24 09:50 | 골드 조정 (4막·종막·세르카) | 필요 |
| **B** | 6/24 09:50 (A와 동시) | 4막·종막·세르카 싱글모드 추가 | 필요 |
| **C** | 7/8 | 신규 딜러 **차원술사** | 불필요 |
| **D** | 8/5 | 신규 레이드 **벨가르딘**(8인) + 나메 플래그 | 필요 |

> 세트 A·B는 같은 시점이라 함께 활성화·배포한다.
> 세트 D는 `availableFrom:'2026-08-05'`로 8/5 전엔 자동 숨김(실수로 먼저 배포돼도 안전).

---

## 세트 A — 골드 조정 (`src/data/raids.ts`, 마커 `UPDATE-A`)

활성화: 각 난이도의 **활성(기존값) 줄을 주석 처리**하고 바로 아래 **주석(신규값) 줄을 해제**.

| 레이드 | 기존 → 신규 |
|---|---|
| ACT4 NORMAL (1700) | 16500/16500 → **13500/13500** |
| ACT4 HARD (1720) | 42000 → **38000** |
| FINAL NORMAL (1710) | 20000/20000 → **16000/16000** |
| FINAL HARD (1730) | 52000 → **48000** |
| SERKA NORMAL (1710) | 17500/17500 → **16000/16000** |

## 세트 B — 싱글모드 추가 (`src/data/raids.ts`, 마커 `UPDATE-B`)

값 = 각 레이드 노말(6/24 신규값). 활성화 2단계:
1. `RaidId` 유니온의 `UPDATE-B` 3줄(`ACT4_SINGLE`·`FINAL_SINGLE`·`SERKA_SINGLE`) 주석 해제.
2. ACT4·FINAL·SERKA `difficulties[]`의 `UPDATE-B` SINGLE 줄 주석 해제.
   - ACT4_SINGLE 1700 / 13500 / 13500
   - FINAL_SINGLE 1710 / 16000 / 16000
   - SERKA_SINGLE 1710 / 16000 / 16000 (세르카는 매칭이지만 '싱글'로 표시)

> 세트 A+B 활성화 후 **반드시** `npm run gen:worker-raids` → `cloudflareworker.js`의 `GEN:RAID-LEDGER` 블록 교체.

## 세트 C — 차원술사 (`src/constants.ts`, 마커 `UPDATE-C`)

`JOB_OPTIONS`에서 `'가디언나이트'` 뒤에 콤마를 추가하고 `// , '차원술사'` 주석 해제.
SUPPORT_CLASSES에 없으므로 자동 **DPS**. 워커 무관.

## 세트 D — 벨가르딘 (`src/data/raids.ts` + `CharacterFormList.tsx`, 마커 `UPDATE-D`)

8인·character·주간·top3 포함. 나이트메어는 `belgardinNightmare` 플래그(체크 시 나메, 아니면 하드).
값: 노말 1750/50000, 하드 1770/62000, 나메 1780/75000 (전부 일반골드). `availableFrom:'2026-08-05'`.

활성화:
1. `raids.ts` `RaidFamily`의 `UPDATE-D`(`'BELGARDIN'`) 주석 해제.
2. `raids.ts` `RaidId`의 `UPDATE-D` 3줄(`BELGARDIN_NORMAL/HARD/NIGHTMARE`) 주석 해제.
3. `raids.ts` `RAIDS` 배열의 `UPDATE-D` 벨가르딘 객체 블록 주석 해제.
4. `CharacterFormList.tsx`의 `UPDATE-D` 벨가 나메 체크박스 JSX 블록 주석 해제.
5. `npm run gen:worker-raids` → 워커 GEN 블록 교체.

> 플래그 배선(`Character.belgardinNightmare`, `RequiresFlag`, `EligibilityContext`, 폼 행/기본값)은
> 이미 라이브로 들어가 있음(불활성, 빌드 무해). D 활성화는 위 5단계만.

---

## 활성화 공통 절차

1. 해당 세트 마커의 주석 해제(A는 활성↔주석 줄 맞바꿈).
2. raids.ts가 바뀌는 세트(A·B·D)는 `npm run gen:worker-raids` 실행 → 출력으로 `cloudflareworker.js`의
   `// === GEN:RAID-LEDGER-BEGIN ===` ~ `// === GEN:RAID-LEDGER-END ===` 블록 교체.
3. `npx tsc --noEmit` · `npm run build` 통과 확인.
4. `npm run deploy` (사이트). 워커 바뀐 세트(A·B·D)는 `cloudflareworker.js`를 **수동 재배포**.
5. 커밋·푸시.

## 검증 팁
- 시간 게이팅 헬퍼(`getEligibleRaids` 등)는 `now` 인자를 받으므로, 배포 전 콘솔/테스트에서
  `getEligibleRaids(ctx, new Date('2026-08-05T01:00:00Z'))` 식으로 시뮬레이션 가능.
- 벨가르딘은 8/5 이전 `now`로는 안 나오고, 이후 `now`로 노말/하드/나메가 플래그·아이템레벨대로 나와야 함.
