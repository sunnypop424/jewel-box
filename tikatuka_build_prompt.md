# 티카투카(Tikatuka) 보드게임 시뮬레이터 — 구현 프롬프트

당신은 로스트아크 여름 이벤트 보드게임 **'티카투카'**를 웹으로 구현합니다.
아래 규칙을 **하나도 빠짐없이** 정확히 구현하세요. 임의로 규칙을 바꾸거나 단순화하지 마세요.
한 가지라도 모호하면 임의로 정하지 말고, 이 문서의 "구현 결정사항(가정)" 섹션의 값을 따르세요.

---

## 1. 게임 개요

- 플레이어(사람) vs **AI 컴퓨터** 1:1 턴제 보드게임.
- 주사위(1~6)를 한 턴에 하나씩 굴려 보드에 배치하며, **3개 라인 중 2개 이상의 라인에서 합으로 이기면 승리**.
- AI는 ★0 ~ ★5 의 6단계 실력으로 나뉜다(자세한 동작은 §8).

---

## 2. 보드 구조 & 상태 모델

- 보드에는 **라인이 총 3개**(Line 0, 1, 2).
- 각 라인은 **왼쪽 진영(상대 필드) / 오른쪽 진영(내 필드)** 로 나뉜다. (좌우 배치는 UI 자유)
- 따라서 필드는 총 6개: `{ line: 0|1|2, side: 'me'|'opponent' }`.
- **각 필드에는 주사위를 최대 3개까지** 놓을 수 있다.
- 각 주사위 객체: `{ value: 1~6, shield: boolean, owner: 'me'|'opponent' }`.

권장 상태 형태:
```
state = {
  lines: [
    { me: [die...], opponent: [die...] },  // 0
    { me: [die...], opponent: [die...] },  // 1
    { me: [die...], opponent: [die...] },  // 2
  ],
  turn: 'me' | 'opponent',
  tajja: { me: true, opponent: true },   // 타짜 손놀림 사용 가능 여부(1회)
  hold:  { me: false, opponent: false }, // 홀드 여부
  tikatuka: { me: false },               // 티카투카 발동 여부
  phase: 'playing' | 'ended',
}
```

## 3. 핵심 규칙 (확정 — 절대 누락 금지)

### 3.1 승리 판정
1. 각 라인마다 `내 필드 주사위 합` vs `상대 필드 주사위 합` 을 비교한다.
   - 내 합이 더 높으면 그 라인은 **내 승리 라인**.
2. **3개 라인 중 2개 이상에서 이기면 승리.**
3. 결과가 **1승 1패 1무**인 경우 → **전체 라인의 내 주사위 총합 vs 상대 주사위 총합** 이 높은 쪽이 최종 승리. (총합도 동점이면 무승부 처리)

### 3.2 밀어내기(공격)
4. 주사위를 굴려 나온 값이 **같은 라인의 상대 필드**에 존재하는 값과 같으면, 그 상대 주사위를 **밀어낼 수 있다(선택 사항, 강제 아님)**.
   - 밀어내기는 **같은 라인 내에서만** 가능. (다른 라인의 같은 숫자는 못 민다)
   - 한 필드에 같은 숫자가 여러 개 있으면 **주사위 하나로 그 숫자 전부**를 민다.
5. 밀어낼 때 **내가 굴린 그 주사위도 사라진다**(필드에 남지 않음). 동시에 **'쉴드' 주사위를 정확히 1개만 추가로 제공**받는다.
   - **쉴드는 밀어내기 1회당 단 1개.** 그 쉴드를 배치하고 나면 다음 턴부터는 다시 **일반 주사위**가 굴려진다.
   - 다시 밀어내기에 성공하면, 그때 또 쉴드 1개가 제공된다. (밀어낼 때마다 1개씩)
6. 밀려난 상대 주사위는 **완전히 사라진다**(상대가 재배치 불가).

### 3.3 쉴드(Shield)
7. 추가 제공되는 주사위는 **'쉴드'**가 붙어 있고, **내 필드/상대 필드 어디든** 배치 가능하다.
   - 가이드(강제 아님): **값 1~3이면 상대 필드, 4~6이면 내 필드**로 보내는 게 유리.
   - 내 필드에 둔 쉴드(높은 값)는 내 합을 올린다.
   - 상대 필드에 둔 쉴드(낮은 값)는 **상대 합에 더해지지만**, 상대의 슬롯 하나를 막아 **상대가 그 칸에 높은 수를 못 넣게 캡을 씌우는 용도**다.
8. **필드에 놓인 쉴드 주사위는 밀어낼 수 없다.** (영구 고정)
9. 쉴드 주사위는 **배치만으로 밀어내기를 발동하지 않는다**(밀어내기는 오직 '굴려서 매칭'했을 때만).

### 3.4 필드 잠금
10. **어느 쪽이든 자기 필드에 주사위 3개가 다 차면, 그 라인에서는 굴려서 매칭이 나와도 상대를 밀어낼 수 없다.** (플레이어·AI 모두 대칭 적용)
    - 자기 필드가 가득 찬 라인 = **그 필드의 주인만** 그 라인에서 공격력(밀어내기)을 잃는다.
    - 즉, **내 필드가 꽉 차면 나만** 그 라인 공격 불가(AI는 여전히 내 비-쉴드 주사위를 밀 수 있음). **AI 필드가 꽉 차면 AI만** 그 라인 공격 불가(나는 여전히 AI 주사위를 밀 수 있음).

### 3.5 게임 종료
11. **모든 필드(6개)가 다 차면(각 3개)** 게임 종료 → 승부 판정.
12. 단, **플레이어가 홀드**하면 플레이어는 더 안 던지므로, 이 경우 **AI 필드가 다 차는 순간** 최종 비교한다(§5.2 참고). (AI는 홀드하지 않는다)

### 3.6 선공 보정 — 첫 주사위 쉴드
13. **선공(먼저 두는) 측의 첫 주사위는 쉴드 주사위로 시작한다.**
    - 게임 첫 턴은 보드가 비어 있어 밀어내기가 불가능하므로, 선공은 이 쉴드를 **자기 필드에 배치**(밀어낼 수 없는 보호 주사위)한다.
    - **후공의 첫 주사위는 일반 주사위.** 이후 양측 모두 일반 주사위로 진행하며, 쉴드는 §3.2의 밀어내기로만 추가 획득한다.
    - 이 보정은 선공의 이점을 후공의 "정보 우위(상대 배치를 보고 둠)"와 균형 맞추기 위한 것.

## 4. 턴 진행

- **굴리기 버튼은 없다.** 상대 턴이 끝나면 **내 주사위가 자동으로 굴려져서** 화면에 나타난다(턴 시작 시 자동 굴림).
- **선공의 첫 턴**: 자동으로 굴려진 주사위가 **쉴드 주사위**로 나온다(§3.6). 나머지 흐름은 동일.
- 한 턴 = **자동으로 굴려진 주사위 1개를 처리**하는 것.
- 처리 절차:
  1. (선택) **타짜의 손놀림** 버튼을 누름 → **즉시 주사위가 하나 더 굴려져** 나온다(별도 굴리기 동작 없음). 처음 굴린 것과 새로 굴린 것 **둘 중 하나를 선택**해 사용하고, 안 쓴 건 버린다.
  2. (사용할) 주사위를 다음 중 하나로 처리:
     - **(a) 내 필드에 배치**: 라인 선택, 그 라인 내 필드에 빈 슬롯이 있어야 함 → 내 합 증가, 턴 종료.
     - **(b) 밀어내기**: 같은 라인 상대 필드에 매칭값 존재 + (내 필드가 그 라인에서 가득 차 있지 않아야 함, §10) → 상대 주사위 제거 + 내 주사위 소멸 + **쉴드 1개 획득 → 즉시 쉴드 배치** → 턴 종료.
  3. 턴을 상대(AI)에게 넘긴다. AI가 처리를 마치면 다시 내 주사위가 자동으로 굴려진다.
- 일반(비쉴드) 주사위는 **내 필드에만** 배치한다(상대 필드에는 쉴드만 갈 수 있다).
- 굴린 주사위 값은 고정, **배치할 라인은 플레이어가 선택**한다.

## 5. 특수 기능

### 5.1 타짜의 손놀림
- 턴 시작 시 자동으로 굴려진 주사위가 맘에 안 들면, **타짜의 손놀림 버튼을 누르면 즉시 주사위가 하나 더 굴려진다**(별도 굴리기 동작 불필요). 처음 것은 그대로 둔 채 둘 중 하나를 선택해 사용, 안 쓴 주사위는 버려진다.
- **게임 전체에서 1회**, 그리고 **승리가 확정되기 전까지만** 사용 가능.
- 상대 AI는 **★3 이상일 때만** 이 기능을 사용한다.

### 5.2 홀드(Hold) — 플레이어 전용
- **홀드는 플레이어만 사용 가능하다. AI는 홀드를 사용하지 않는다.**
- 홀드하면 **자신은 더 이상 주사위를 던지지 않는다**(현재 배치로 고정).
- **상대(AI)는 계속 던진다.**
- 홀드 중에도 **AI는 내 비-쉴드 주사위를 계속 밀어낼 수 있고, 나는 대응할 수 없다.**
- 플레이어가 홀드한 경우 **AI 필드가 다 차는 순간** 최종 비교(종료). 플레이어가 홀드하지 않으면 6개 필드가 모두 차면 종료.

### 5.3 티카투카(Tikatuka)
- **내 턴에 아무 때나** 누를 수 있다.
- **패널티 없음.** 다른 기능을 잠그지 않는다(타짜와 완전히 독립).
- 누른 뒤에도 플레이는 **끝까지 계속**된다.
- **승리할 경우 추가 승점(보너스)을 얻는다.** (순수 보너스 개념)
- 결론적 전략: 패널티가 없으므로 **이긴다고 판단되는 순간 바로 누르는 것이 항상 이득**.

## 6. AI 난이도 ★0 ~ ★5

공통: **AI는 홀드와 티카투카를 사용하지 않는다**(둘 다 플레이어 전용). **타짜는 ★3 이상만 사용.** AI는 자기 필드가 다 찰 때까지 계속 던진다.

- **★0 (랜덤)**: 완전 무작위. 굴린 주사위를 임의 라인 내 필드에 배치. 밀어내기/쉴드 배치 모두 무작위. 홀드·타짜 미사용. 라인 배분 개념 없음.
- **★1 (하수)**: 탐욕적 자기합 극대화(예: 합이 가장 낮은 내 라인에 채움). **명백한 밀어내기**(상대 최고합 라인에서 매칭)만 가끔 수행. 쉴드 가이드 약하게 적용. 타짜 미사용.
- **★2 (중수)**: **"2라인 승리" 목표를 인지**. 가망 없는 1라인을 의도적으로 포기하고 나머지 2라인에 집중. 쉴드 가이드(1~3 상대/4~6 자기) 적용. 밀어내기로 접전 라인 흔들기. 타짜 미사용.
- **★3 (상수)**: **타짜의 손놀림 사용 시작**(접전·결정적 순간). 상대 합 캡(쉴드를 상대 필드에) 의식. **필드 잠금(내 3칸 채우기) 타이밍** 활용. 밀어내기의 가치(템포 손해 vs 이득)를 계산.
- **★4 (고수)**: **기대값 기반 의사결정.** 라인 배분 최적화, **동점 시 전체 총합 타이브레이커까지 고려.** 상대의 밀어내기 위협을 읽고 방어적으로 쉴드/필드잠금 운용. 타짜를 가치가 가장 큰 순간에 사용. 남은 주사위·슬롯을 카운팅.
- **★5 (마스터)**: 가능한 수를 휴리스틱 + **얕은 시뮬레이션/몬테카를로**로 부분 탐색하여 거의 최적 플레이. 타짜·쉴드·밀어내기 전부 EV 최적화. 타이브레이커와 종료 시점을 역산. ★4보다 명확히 강해야 함.

각 단계는 바로 아래 단계 대비 **체감되는 실력 차이**가 있어야 한다.

## 7. UI/UX 요구사항

- **모달이 아니라 독립 페이지로 제공한다**(라우트 `/tikatuka`). 사이드바 "게임" 메뉴에서 진입.
- **모바일 대응 필수.** 좁은 화면에서도 3+3 주사위가 한 줄에 들어오도록 주사위 크기를 `clamp()`로 화면 폭에 맞춰 축소하고, 라인 정보/여백도 모바일에서 줄인다.
- 3개 라인 × (내 필드 / 상대 필드)를 한눈에 보이게 배치. 각 필드의 슬롯(최대 3) 시각화.
- 각 라인별 **현재 합과 승/패/무 상태**를 실시간 표시. 전체 총합도 표시.
- 주사위는 1~6 pip(또는 숫자)로, **쉴드 주사위는 명확히 구분**(테두리/아이콘 등).
  - 주의: pip을 CSS로 부모 크기에 비례시키는 방식(`aspect-ratio`, 또는 그리드 셀 기준 `%` 높이)은 프로젝트의 Tailwind preflight 등과 만나면 셀 안에서 높이가 0으로 무너져 **눈이 안 보이는** 버그가 난다. **pip은 SVG `<circle>`(`viewBox="0 0 100 100"`, svg는 `width/height:100%`)로 그려라** — 어떤 크기(고정 px / `clamp` 클래스)에서도 정확히 스케일되고 CSS 환경에 영향받지 않는다.
- **굴리기 버튼 없음.** 내 턴이 시작되면 주사위가 자동으로 굴려져 화면에 나타난다. 흐름: (자동 굴림) → (타짜 선택지) → 배치/밀어내기. 밀어내기 가능 슬롯은 하이라이트.
- **주사위 연출**: 굴림이 멈출 때 **살짝 튀며 잠깐 멈춰 결과를 보여주고**, 배치는 **위에서 내려놓는 애니메이션**, 밀어내기는 **바깥으로 튕겨 날아가는 애니메이션**을 준다. (`prefers-reduced-motion`이면 즉시 처리)
- 버튼: **타짜의 손놀림(1회, 사용 후 비활성) / 홀드 / 티카투카**. 사용 불가 조건이면 비활성 + 이유 툴팁. (굴리기 버튼은 두지 않는다)
- **AI 난이도 선택 UI(★0~★5).**
- 게임 종료 시 라인별 결과 + 타이브레이커 적용 여부 + 티카투카 보너스 반영 결과를 명확히.
- 로그/리플레이(어떤 주사위가 어디서 밀렸는지) 있으면 디버깅·검증에 좋음.

## 8. 기술 스택 (제안 — 변경 가능)

- React + Vite + TypeScript, 상태는 단일 reducer로 관리(규칙 엔진과 UI 분리).
- **규칙 엔진을 순수 함수 모듈로 분리**(`engine.ts`)하고, AI도 그 엔진의 합법수(legal moves) 생성기를 공유하게 하라. 이래야 규칙 누락 없이 검증 가능.
- 단일 HTML로도 가능하면 무방.

## 9. 구현 결정사항(가정) — 규칙이 명시 안 한 부분은 이대로 구현

이 항목들은 원 룰에 100% 명시되진 않았으나, 일관된 게임을 위해 다음과 같이 확정한다. (사용자가 다르게 지정하면 그 값을 우선)

- **선공**: **무작위(코인토스)** 로 결정. **선공의 첫 주사위는 쉴드 주사위**(§3.6).
- **턴 교대**: 한 턴에 주사위 1개 처리 후 상대 턴.
- **AI는 홀드/티카투카를 사용하지 않는다**(둘 다 플레이어 전용). AI는 자기 필드가 다 찰 때까지 계속 던진다.
- **쉴드 주사위 값**: 밀어내기로 받는 쉴드 주사위는 **1~6 무작위로 굴려서** 값이 정해진다(그래서 1~3/4~6 배치 가이드가 의미를 가진다). 받은 즉시 같은 턴에 배치한다.
- **밀어내기 후 추가 행동**: 밀어내기 + 쉴드 배치까지가 **한 턴**. 추가 굴리기 없음.
- **일반 주사위는 내 필드에만**, 쉴드만 상대 필드 배치 가능.
- **합법수 없음 처리**: 던지는 쪽이 둘 곳이 전혀 없고 밀어내기도 불가하면 자동 홀드 처리.
- **티카투카는 플레이어 전용 보너스**로 구현(AI는 점수 개념이 없어 사용하지 않음).
- 동점(라인 합 동일)은 **무(draw) 라인**으로 처리.

## 10. 누락 방지 체크리스트 (구현 후 반드시 자가 검증)

- [ ] 3라인 중 2라인 승리로 승패 결정, 1승1패1무는 전체 총합 타이브레이커
- [ ] 밀어내기는 **같은 라인**에서만, 매칭 시 그 숫자 **전부** 제거
- [ ] 밀어내면 내 굴린 주사위 소멸 + **쉴드 정확히 1개** 획득(밀어내기 1회당 1개, 그 뒤엔 다시 일반)
- [ ] 밀려난 주사위는 영구 소멸(재배치 불가)
- [ ] 쉴드는 밀리지 않음(영구 고정)
- [ ] **선공의 첫 주사위는 쉴드로 시작**(후공 첫 주사위는 일반), 첫 턴 보드가 비어 자기 필드에 배치
- [ ] 쉴드는 내/상대 필드 모두 배치 가능, 상대 필드 쉴드는 상대 합에 더해지되 슬롯 캡 역할
- [ ] **굴리기 버튼 없음** — 턴 시작 시 자동 굴림, 타짜 누르면 즉시 추가 굴림
- [ ] 필드 3개 가득 = 그 라인에서 **그 필드 주인만** 밀어내기 불가 (**플레이어·AI 대칭 적용**)
- [ ] 모든 필드(6개) 충원 시 종료, 플레이어 홀드 시 AI 필드 충원 시점에 종료
- [ ] 타짜: 전체 1회, 승리 확정 전, AI는 ★3+만
- [ ] 홀드: **플레이어 전용**(AI 사용 불가), 본인만 멈춤, AI는 계속 던지고 내 비-쉴드 주사위를 밀 수 있음
- [ ] 티카투카: 내 턴 아무 때나, 패널티 없음, 잠금 없음, 승리 시 보너스
- [ ] ★0~★5 실력 단계가 체감되게 구분
